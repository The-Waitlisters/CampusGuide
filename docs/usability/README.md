# Usability Testing Infrastructure

This directory contains all documentation and infrastructure prepared for conducting usability testing of the CampusGuide mobile application.

The usability testing framework was designed and implemented during Sprint 3 to prepare for pilot testing in Sprint 4 and full-scale usability evaluation in Release 3.

---

## Directory Contents

### 1. Pilot_Usability_Test_Script.md

Contains the standardized usability testing procedure, including:

- Ethical consent statement
- Participant information section
- Structured task scenarios
- Success criteria
- Moderator logging instructions

This script ensures consistency across all testing sessions.

[Pilot Usability Test Script](https://docs.google.com/document/d/1yYAQyc5awz9wWGNG91hpA87dzD3lFasrUEJe848AH1Q/edit?usp=sharing)

---

### 2. SUS_Form_Description.md

Documents the implementation of the System Usability Scale (SUS) questionnaire, including:

- Overview of the SUS methodology
- Scoring formula
- Interpretation guidelines
- Usage plan within the project

The actual survey is implemented using Google Forms.

[SUS Google Form](https://forms.gle/f6EV6xrdPWFVfQ327)

---

### 3. Logging_Template.md

Describes the structured data logging methodology used during usability sessions, including:

- Quantitative metrics collection
- Duration calculation formula
- Error tracking
- Task completion recording
- Summary metric calculations

A Google Sheets template is used for live data collection.

[Logging Sessions Data](https://docs.google.com/spreadsheets/d/1UpLCTWNnXDsdWjqvmPQhV0EWbOWe4dIq115kJqZ2T3U/edit?usp=sharing)

---

## Purpose

The goal of this infrastructure is to ensure:

- Structured and reproducible usability testing
- Ethical data collection practices
- Quantitative and qualitative usability measurement
- Data-driven UI improvement decisions

This setup enables pilot usability testing in Sprint 4 and comprehensive analysis in Sprint 5.
